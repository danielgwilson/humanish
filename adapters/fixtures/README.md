# Adapter Fixtures

These fixtures are public-safe contract packets for adapter parity work.

They intentionally use synthetic product surfaces:

- `post-auth-return-dry-run` proves web-app route, persona, scenario, and
  milestone ownership without changing core run-bundle schemas.
- `terminal-feedback-lifecycle` proves terminal evidence, feedback draft, cost
  policy, and redaction lifecycle without relying on browser-only assumptions.

Do not replace these fixtures with real product transcripts, screenshots,
credentials, customer data, hosted stream URLs, or private artifact paths.
